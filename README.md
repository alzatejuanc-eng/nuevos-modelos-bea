## 🤖 Los 8 modelos de negocio propuestos

### Horizonte 1 — CORE (6–12 meses · Riesgo bajo)

**01 · BEA AI Operations Hub**
Capa de IA predictiva sobre los datos existentes de MTB y AVA. Motor de predicción de demanda por corredor, mantenimiento predictivo de flota y reportes ejecutivos generados por LLM. Modelo SaaS por número de unidades monitoreadas.

**02 · BEA Urban Data Exchange**
Monetización de 35 años de datos de movilidad urbana en LatAm mediante APIs anonimizadas dirigidas a gobiernos, empresas inmobiliarias, aseguradoras y startups de logística.

---

### Horizonte 2 — ADYACENTES (12–24 meses · Riesgo moderado)

**03 · BEA MaaS Orquestador**
Plataforma de Mobility as a Service que integra BRT, taxi, bicicleta, scooter y metro en un único wallet de movilidad con journey planner multimodal potenciado por IA. Ciudad piloto sugerida: Guayaquil (Metrovía).

**04 · BEA Smart City Command Center**
Centro de mando integrado para secretarías de movilidad con IA de visión computacional, Digital Twin de la red de transporte y alertas en tiempo real. Modelo contractual PPP plurianual.

**05 · BEA Transit Tech Academy**
Plataforma educativa EdTech con tutor IA adaptativo para la formación de operadores, conductores, técnicos y funcionarios públicos. Certificación BEA en gestión de transporte inteligente.

---

### Horizonte 3 — DISRUPTIVOS (24–36 meses · Recompensa exponencial)

**06 · BEA MovilWallet — El banco del pasajero**
Wallet digital de movilidad con micro-crédito para población no bancarizada, scoring crediticio basado en comportamiento de viaje y red de pagos en comercios cercanos a paradas BEA.

**07 · BEA AutoFleet — Flota como Servicio (RaaS)**
BEA asume la gestión operativa completa de redes de transporte mediante un sistema multiagente de IA autónoma. Cobra porcentaje del recaudo total gestionado más bono de desempeño por KPIs.

**08 · BEA Transit AI Marketplace** *(Moonshot)*
Plataforma abierta donde startups y empresas construyen sobre la infraestructura de datos y APIs de BEA. Incluye el desarrollo de un modelo fundacional de movilidad urbana para LatAm — el "GPT de la movilidad".

## 🏗️ Metodología aplicada

Este análisis sigue los principios de **OpenExO Sprint**, el framework de innovación exponencial de Salim Ismail, adaptado con el marco **MIEDE 2.0** de Transformación ExO:

- **Horizonte 1 (Core):** Optimizar y escalar el modelo actual con IA
- **Horizonte 2 (Adyacente):** Expandir hacia nuevos clientes y mercados con capacidades existentes
- **Horizonte 3 (Disruptivo):** Crear nuevos mercados o disrumpir el propio modelo de negocio actual

Los modelos fueron evaluados en 4 dimensiones: tiempo al mercado, potencial de ingreso, complejidad de ejecución y componente de inteligencia artificial.

### Ver el informe localmente

El informe es un archivo HTML estático autocontenido. Descarga `index.html` y ábrelo en cualquier navegador moderno — no requiere servidor ni dependencias.

### Desplegar en Vercel ✅ (recomendado)

> **Importante:** Vercel requiere que el archivo principal se llame `index.html` en la raíz del repositorio. Este repositorio ya está configurado correctamente.

```bash
# 1. Clonar el repositorio
git clone https://github.com/tu-usuario/bea-exo-sprint.git
cd bea-exo-sprint

# 2. Conectar en vercel.com → "Add New Project" → importar desde GitHub
# 3. En la configuración de Vercel:
#    Framework Preset: Other
#    Root Directory: ./  (dejar en raíz)
#    Build Command: (dejar vacío)
#    Output Directory: ./  (dejar en raíz o punto)

# 4. Click en Deploy — el informe queda en:
# https://bea-exo-sprint.vercel.app
```

Si el error persiste en Vercel, agrega un archivo `vercel.json` en la raíz:

```json
{
  "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
}
```
